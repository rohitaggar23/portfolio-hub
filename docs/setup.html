GitHub: From Zero → Polished (Step-by-step)

Rule of thumb: People reviewing your GitHub care about proof. Your repos already include outputs, logs, plots, traces, and reports. Your job is to publish them cleanly and make them easy to find.

0) What you have

You downloaded a bundle containing separate repos:

  • atlas-rag-copilot/
  • sentinel-eval-safety-suite/
  • orion-tool-agent/
  • rescue-swarm-ai/
  • restaurant-hybrid-recommender/
  • portfolio-hub/ (landing page)

1) Create your GitHub account + basics

  • Add a professional profile photo.
  • Add a one-line headline like: AI/ML Engineer • GenAI Systems • RAG • Agents • Recommenders.
  • Add location + email.

2) Create 6 empty repositories

Create these repos (public):

  1. atlas-rag-copilot
  2. sentinel-eval-safety-suite
  3. orion-tool-agent
  4. rescue-swarm-ai
  5. restaurant-hybrid-recommender
  6. portfolio-hub

Important: do not initialize with README/license on GitHub (you already have them locally).

3) Push each repo (repeat per folder)

Example for atlas-rag-copilot:

cd atlas-rag-copilot

git init
git add .
git commit -m "Initial commit: project with outputs"

git branch -M main
git remote add origin https://github.com/<your-username>/atlas-rag-copilot.git
git push -u origin main

Repeat the same for the other projects (change folder name and remote URL).

4) Deploy the landing page (GitHub Pages)

  1. Go to portfolio-hub repo → Settings → Pages
  2. Source: Deploy from a branch
  3. Branch: main, Folder: /docs
  4. Save

Your portfolio URL will be:

https://<your-username>.github.io/portfolio-hub/

5) Personalize the landing page

Edit:

portfolio-hub/docs/data/config.json

Replace <your-username>, LinkedIn, email. Commit and push.

6) Polish checklist

  • Repo About panel: good description + topics + link to your landing page.
  • Pin repos: pin portfolio-hub, atlas-rag-copilot, and 2–3 others.
  • Profile README: create a repo named <your-username> with a strong README that links to the landing page.
  • Releases: tag a release for each repo (optional but looks senior).

7) Final verification (simulate reviewer behavior)

  1. Open the landing page in an incognito window.
  2. Click each repo from the landing page.
  3. Confirm: README loads, images load, outputs exist.
You’re done when:
Your landing page loads fast, every project has a crisp README, and outputs are visible without running anything.
